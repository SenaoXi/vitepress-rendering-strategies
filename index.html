Cross-framework Islands Architecture for VitePress

Get Started
View on GitHub
🏝️

Islands Architecture

Inspired by Astro's Islands Architecture. Each component container completes hydration independently with framework isolation, avoiding global state conflicts.

🎯

Flexible Rendering Strategies

Four rendering modes - ssr:only (default), client:only, client:load, and client:visible. Optimize for performance and user experience based on component criticality.

⚡

SPA Navigation Optimization

spa:sync-render directive eliminates component flicker during route transitions by synchronizing pre-rendered HTML injection with Vue's rendering cycle.

🚀

Static-First with Progressive Enhancement

SSG-first architecture with build-time pre-rendering. Components are pre-rendered at build time, with selective client-side hydration only where interaction is needed.

🛠️

Development Excellence

Full HMR support, consistent dev/prod behavior, and TypeScript integration. Environment consistency prevents production surprises.

📦

Production Ready

Full MPA mode compatibility and Vue-to-React prop initialization via rendering container. Seamless integration with VitePress production builds.
